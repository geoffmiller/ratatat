What this file is
-----------------
A quick orientation file for coding assistants and automation agents working in this repo.
Read it before making changes so behavior stays consistent with project expectations.

Project at a glance
-------------------
Ratatat is an Ink-compatible terminal UI runtime with:
- a React adapter,
- a framework-agnostic core API,
- and a native Rust diff engine.

Preferred import paths
----------------------
Use explicit entrypoints in docs/examples unless compatibility is needed:

- React adapter: `ratatat/react`
- Core/raw runtime: `ratatat/core`
- Scene/layout internals: `ratatat/scene`
- Native binding only: `ratatat/native`
- Root `ratatat` import remains compatibility-friendly, but not preferred in new docs.

Runtime modes
-------------
1) React mode (`ratatat/react`)
- Use `render()`, `<Box>`, `<Text>`, hooks, etc.
- Main examples: `./examples/`

2) Raw-buffer mode (`ratatat/core`)
- Paint directly into `Uint32Array` via `Renderer`.
- Main examples: `./examples-raw/`
- API details: `./docs/raw-buffer.md`

3) Inline mode
- React inline: `renderInline()` from `ratatat/react`
- Core inline: `createInlineLoop()` from `ratatat/core`

---------------------------

Common commands
---------------
Install deps:
- npm install

Build:
- npm run build:ts
- npm run build

Test:
- npm test

Benchmarks:
- npm run bench:ts
- npm run bench:startup

Run examples
------------
- React mode: `node --import @oxc-node/core/register examples/kitchen-sink.tsx`
- React mode: `node --import @oxc-node/core/register examples/stress-test.tsx`
- Raw-buffer mode: `node --import @oxc-node/core/register examples-raw/ascii-3d.ts`
- Raw-buffer mode: `node --import @oxc-node/core/register examples-raw/conway.ts`
